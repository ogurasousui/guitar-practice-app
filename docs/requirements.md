# Guitar Practice App 要件定義

## 目的

ギター練習中に、ドラムとベースの簡単なバッキングを流しながら、画面上に表示される短いギターフレーズのタブ譜を見て練習できるWebアプリを作る。

ユーザーは複数のフレーズ候補から好きなものを選び、リズムに合わせて反復練習する。

## 公開形態

- GitHub Pagesで公開できる静的Webアプリにする
- ログイン不要
- サーバー不要
- PCブラウザ、スマートフォンブラウザの両方で閲覧できる
- 初期版ではユーザーデータ保存や課金機能は持たない

## 想定ユーザー

- ギター初心者から中級者
- 1から2小節程度の短いフレーズを反復練習したい人
- メトロノームだけではなく、ドラムとベースの伴奏に合わせて練習したい人
- タブ譜を見ながら耳と手を同時に慣らしたい人

## 決定事項

MVPでは以下の方針で実装する。

- 音源方式: Web Audio APIで簡易ドラム・ベースを生成する
- 初期キー: A minor
- 初期スケール: A minor pentatonic
- 拍子: 4/4
- 初期BPM: 90
- BPM範囲: 60から160
- 初期フレーズ数: 6個
- アプリ表示名: Guitar Practice
- 内部プロジェクト名: guitar-practice-app
- 技術構成: Vite + TypeScript + React
- 公開方式: GitHub Pages

## MVPの機能

### 1. バッキング再生

ドラムとベースのバッキングをループ再生できる。

必要な操作:

- 再生
- 停止
- テンポ変更
- ループ再生

初期実装では、実音の録音ファイルではなく、ブラウザ上でドラムとベースの音を生成する。

### 2. タブ譜表示

画面上に、1から2小節程度のギターフレーズのタブ譜を複数表示する。

表示する情報:

- フレーズ名
- 難易度
- 推奨テンポ
- タブ譜
- 拍や小節の区切り
- 可能であれば簡単な演奏メモ

例:

```text
Phrase A - Minor Pentatonic
e|----------------5-8-|
B|------------5-8-----|
G|--------5-7---------|
D|----5-7-------------|
A|5-7-----------------|
E|--------------------|
```

### 3. フレーズ一覧

複数の短いフレーズを同時に表示する。

ユーザーは特定のフレーズを選択しなくても、画面を見ながら好きなフレーズを練習できる。

初期表示数:

- PC: 4から8個程度
- スマートフォン: 縦スクロールで複数表示

### 4. テンポ調整

ユーザーが練習しやすいようにBPMを変更できる。

初期値:

- BPM 90

設定範囲:

- BPM 60から160

### 5. キー・スケールの初期固定

MVPでは複雑なキー変更は入れず、まずは1つのキーに固定する。

初期設定:

- A minor
- A minor pentatonic
- 4/4拍子

## 画面構成

### メイン画面

画面上部:

- アプリ名
- BPM表示
- 再生・停止ボタン
- テンポ調整コントロール

中央:

- 現在のバッキング情報
- 小節カウントまたは簡易ビート表示

下部:

- タブ譜カード一覧
- 各カードにフレーズ名、難易度、タブ譜、メモを表示

## データ構造案

フレーズはコード内の配列、またはJSONファイルとして管理する。

```json
{
  "id": "minor-pentatonic-01",
  "title": "Minor Pentatonic 1",
  "difficulty": "easy",
  "key": "A minor",
  "bpm": 90,
  "bars": 1,
  "tab": [
    "e|----------------5-8-|",
    "B|------------5-8-----|",
    "G|--------5-7---------|",
    "D|----5-7-------------|",
    "A|5-7-----------------|",
    "E|--------------------|"
  ],
  "memo": "Aマイナーペンタの上昇フレーズ"
}
```

## 技術方針

### 推奨構成

GitHub Pagesで公開しやすく、軽量に作れる構成にする。

- Vite
- TypeScript
- React
- Web Audio API

理由:

- 静的ファイルとしてビルドできる
- GitHub Pagesに載せやすい
- 音の再生やテンポ制御をブラウザ上で扱いやすい
- 将来的にフレーズ追加やUI改善をしやすい

### 音源方針

MVPではWeb Audio APIで簡易ドラム・ベースを生成する。

メリット:

- 音声ファイルを用意しなくてよい
- BPM変更に強い
- GitHub Pagesに載せやすい

デメリット:

- 生音っぽさは弱い

## MVPで作らないもの

初期版では以下は対象外にする。

- ログイン
- 練習履歴の保存
- 録音
- 採点
- チューナー
- 自動タブ譜生成
- MIDI入力
- 複雑なコード進行編集
- ユーザー投稿

## 将来追加したい機能

- キー変更
- スケール選択
- コード進行選択
- 難易度フィルタ
- フレーズのお気に入り
- 練習履歴
- タイマー
- カウントイン
- ドラムパターン変更
- ベースライン変更
- フレーズ単体の音声再生
- タブ譜のハイライト再生
- PWA対応

## 初期フレーズ案

MVPでは以下のような短いフレーズを用意する。

- Aマイナーペンタ上昇
- Aマイナーペンタ下降
- ブルース風チョーキングなしフレーズ
- 8分音符中心のロックリフ
- 休符を含むリズム練習フレーズ
- 低音弦中心のリフ

## 完了条件

MVPは以下を満たしたら完了とする。

- GitHub Pagesでページを開ける
- 再生ボタンでドラムとベースのバッキングが鳴る
- 停止ボタンで音が止まる
- BPMを変更できる
- 1から2小節のタブ譜が複数表示される
- PCとスマートフォンで表示が崩れない
- ユーザーが画面を見ながらすぐ練習を始められる

## プロジェクト名案

内部プロジェクト名:

- `guitar-practice-app`

表示名候補:

- Guitar Practice
- Fret Practice
- Riff Trainer
- ギター練習帳

初期実装では、リポジトリ名や内部名は `guitar-practice-app`、画面上の表示名は `Guitar Practice` とする。
